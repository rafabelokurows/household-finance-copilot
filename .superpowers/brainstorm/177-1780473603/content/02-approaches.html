Architecture Approaches

Which structure fits Phase 2 best?

A

Simple Single-File App

Structure: One app.py with all logic (tabs, queries, UI)

Pros: Fastest to build, no abstraction overhead, easy to understand flow

Cons: Gets messy as features grow, harder to test, all state global

Best for: Quick MVP, don't anticipate major changes soon

B

Component-Based Architecture

Structure: app.py orchestrates, components/ folder has review_queue.py, browse.py, utils.py

Pros: Clean separation, easier to test, reusable UI components, maintainable

Cons: More files, slight overhead in wiring components together

Best for: Planning Phase 3+ features, want clean boundaries now

C

State Management Layer

Structure: Components + services/ layer (queries, API calls, caching) + session.py for state

Pros: Clear data flow, easy to swap backend, testable, scales to Phase 3+

Cons: More boilerplate, overkill if app stays small

Best for: Long-term evolution, multiple users, complex state
